技术架构

飞书原生优先 + 分层解耦

⚡
性能指标（两层拆分）
内部执行延迟：规则引擎 < 200ms，LLM 推理 < 3s
端到端延迟：规则 < 2s（80%场景），LLM < 5s（20%场景）
说明：端到端包含飞书 Webhook 固有延迟 500ms-2s。异步模式下用户感知 < 3s（先返回"处理中"）。
🛡
风险应对
Agent 幻觉：RAG 强制检索 + Human-in-the-Loop + 置信度阈值
Agent 协作冲突：Orchestrator 统一仲裁 + 预测 Agent > 内容 Agent
LLM 供应商故障：降级模式 + 多供应商备选（百炼/豆包/Claude）
飞书 API 限流：异步队列削峰 + 本地缓存 + 批量写入
📦
Cold Storage 分层
类型A：结构化数据 → 飞书多维表格（Agent 决策日志、行为埋点）
类型B：非结构化数据 → 飞书云文档（对话记录、作品分析、案例拆解）
类型C：归档数据 → 阿里云 OSS（超 6 个月历史数据）
🔗
飞书生态集成矩阵
多维表格：状态存储 / Memory 结构化 / 决策日志 / Pending Pool
云文档：课程内容 / 非结构化记忆 / Skill Document
妙记：直播转录 / 知识点提取 / 题目生成
审批：晋升申请 / 季度评价 / 题目审阅
机器人：Agent 主动推送 / Coach 对话界面
智能伙伴：Coach Agent 底层 LLM / 语义检索
应用引擎：Workflow 编排 / Tool Registry API 封装
新增模块（v4.0→v4.1）

360° 评估 · 等级认证 · 导师激励

📋
季度 360° 评估
新增
权重：自评 25% + 导师 50% + Agent 数据 12.5% + 作品 12.5%

流程：季度初飞书待办推送 → 学员自评问卷 → 导师评价审批流 → Agent 数据自动汇总 → 计算总分 → 季度成长报告 → 触发晋升审核

公式：final_score = self×0.25 + mentor×0.50 + agent×0.125 + work×0.125
🏅
等级认证体系
新增
四级：L1助理 → L2铜牌 → L3银牌 → L4金牌

L2 要求：能力平均 ≥2.0 / 项目 ≥3 个 / 通过季度 Review
L3 要求：能力平均 ≥3.0 / 项目 ≥8 个 / 带教 ≥20h
L4 要求：能力平均 ≥4.0 / 项目 ≥15 个 / 带教 ≥50h

状态校验：能力图谱 = "已发布" 才触发晋升，防止误触发
💎
导师激励积分制
新增
积分规则：辅导 1h +10分 / 预审通过 +2分 / 案例拆解 +20分 / 学员晋升 +50分

兑换：100分 = 1天培训预算 / 500分 = 半天调休 / 1000分 = 年度大奖提名

技术：MentorPointsHistory 子表（替代 JSON），飞书自动化多表关联统计
