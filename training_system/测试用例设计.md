## 1. 简介与范围

### 1.1 文档目的

本文档基于 PRD v2.0 中定义的 Agentic Learning Engine 架构，为 MediaStorm 新人培训与能力图谱系统设计全面的测试用例。覆盖范围包括：

- **核心编排层**：Orchestrator（调度中枢）、Memory（四层记忆架构）、Tool Registry（工具注册表）
- **六大核心 Agent**：诊断 Agent、Coach Agent、内容 Agent、预测 Agent、实战 Agent、AI Assessment Generator
- **工具层（Module 1-5）**：课程库、案例库、资源库、能力图谱、导师网络、测评题库、项目数据、成长路径
- **用户交互层**：学员端飞书消息机器人、导师端预处理看板、管理端洞察仪表盘

### 1.2 测试范围

| 测试类型 | 覆盖范围 | 优先级 |
|---------|---------|--------|
| **功能测试** | 单个 Agent / 模块的独立功能 | P0 |
| **集成测试** | 跨 Agent / 跨模块的协作场景 | P0 |
| **边界测试** | 异常输入、极限条件、容错回退 | P1 |
| **验收测试** | 端到端用户旅程（入职→学习→测评→晋升） | P0 |
| **非功能测试** | 性能、安全、可解释性、数据隐私 | P1 |

### 1.3 术语定义

| 术语 | 测试语境下的定义 |
|------|----------------|
| **Hot Memory** | 每次 Agent 推理时自动注入的冻结快照（USER.md + MEMORY.md） |
| **Cold Recall** | 按需检索的会话档案馆，测试重点在检索准确性和响应时间 |
| **Skill Doc** | 程序性记忆文档，测试重点在自改进逻辑和版本管理 |
| **Pending Pool** | AI 生成测试题的待审队列，测试重点在状态机流转和 Owner 审阅流程 |
| **Human-in-the-Loop** | 需人工审核的关键决策点，测试重点在人机交接的准确性和延迟 |

## 2. 测试策略

### 2.1 功能测试策略

- **黑盒测试**：基于 PRD 中的功能描述和用户故事，验证每个功能点的输入输出是否符合预期
- **状态机测试**：对 Pending Pool、Orchestrator 事件状态、学员风险评级等存在明确状态流转的功能进行全覆盖测试
- **规则引擎测试**：Orchestrator 的决策逻辑包含规则层、策略层、模型层，需分层验证

### 2.2 集成测试策略

- **Agent 间协作测试**：验证 Orchestrator 能否正确调度多个 Agent 并行/串行执行
- **Memory 一致性测试**：验证多个 Agent 读写 Memory 时数据一致性和并发安全
- **Tool Registry 调用链测试**：验证 Agent 调用工具的标准化接口、权限控制和失败回退

### 2.3 边界与异常测试策略

- **输入边界**：空输入、超长输入、特殊字符、并发事件
- **状态边界**：Memory 未初始化、Agent 不可用、LLM API 超时/限流
- **冲突边界**：多个 Agent 给出矛盾建议时的仲裁逻辑

### 2.4 验收测试策略

- **用户旅程端到端测试**：按照 PRD 第七章「用户旅程地图」的三个完整旅程（学员/导师/制片人）执行全流程验证
- **Acceptance Criteria 核对**：逐条验证 PRD 中各模块的 Acceptance Criteria
