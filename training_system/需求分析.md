## 一、项目背景与目标

### 1.1 项目背景

MediaStorm（影视飓风）当前团队规模约 **100人**，年产出 **130+部影片**。随着业务高速增长，团队在人才管理方面面临严峻挑战：

| 痛点 | 现状数据 | 业务影响 |
|------|---------|---------|
| 新人上手周期长 | 剪辑师/编导新人需 **2-3个月** 才能独立产出 | 项目交付延迟，人力成本浪费 |
| 编导岗位淘汰率高 | 年淘汰率约 **10%**（每年约淘汰10人） | 人员流动成本高，团队稳定性差 |
| 知识传承依赖个人 | 优秀编导经验（如Tim的选题逻辑）未系统化沉淀 | 核心员工离职导致经验断层 |
| 派活匹配靠直觉 | 制片凭记忆安排人员，频繁 mismatch | 项目返工率高，客户满意度下降 |
| 晋升标准模糊 | 员工看不到清晰的成长路径 | 人才流失风险，员工动力不足 |

### 1.2 核心范式升级

本系统从传统「人找内容」的 LMS（学习管理系统）升级为 **Agentic Learning（代理式学习）** 架构：

```
v1.0（传统 LMS）：学员 → 打开系统 → 找课程 → 学完 → 做测评 → 等待反馈
                    ↑_____________________________________________↓
                                被动、离散、依赖人工

v2.0（Agentic）：  Agent 感知学员状态 → 自主决策 → 调用工具层 → 主动推送服务
                    ↓                                              ↑
              实时反馈 ← 动态调整 ← Memory 更新 ← 多 Agent 协作
                                主动、连续、智能闭环
```

### 1.3 项目目标

| 目标维度 | 具体目标 | 衡量指标 | 目标值 |
|---------|---------|---------|--------|
| **效率** | 缩短新人独立产出周期 | 从入职到首次独立交付项目 | **2个月 → 2周** |
| **质量** | 降低因"风格不符"导致的人才淘汰 | 编导岗位淘汰率 | **10% → 5%** |
| **传承** | 将优秀员工的经验系统化沉淀 | 建立可复用的"影视飓风方法论" | 案例库覆盖全部爆款 |
| **决策** | 让派活和晋升有据可依 | 项目派活基于能力图谱匹配的比例 | **100%** |
| **导师效率** | 减少资深员工重复带教时间 | 导师基础答疑时间占比 | **30% → 15%** |
| **学习体验** | 提升学员学习完成率与满意度 | 课程完成率 / NPS | **85%+ / >50** |

## 二、用户分析

### 2.1 用户分层

| 用户角色 | 人数（预估） | 核心诉求 | 使用频率 | 关键痛点 |
|---------|------------|---------|---------|---------|
| **新人剪辑师/编导** | 15-20人/年 | 快速学会"影视飓风风格"，独立产出合格作品 | 每天 | 没人系统教、标准模糊、不知离目标多远 |
| **资深剪辑师/编导** | 30-40人 | 成为导师获得成就感，减少重复带教时间 | 每周 | 每次教同样基础内容、新人问题打断创作 |
| **制片人/项目经理** | 5-8人 | 派活时有据可依，知道谁适合做什么 | 每次派活 | 凭印象派活导致 mismatch、改稿多 |
| **HR/培训负责人** | 1-2人 | 追踪新人学习进度，评估培训效果 | 每周 | 数据分散、无法实时掌握全局 |
| **管理层（Tim等）** | 3-5人 | 看到团队整体能力分布，识别人才梯队风险 | 每月 | 缺乏全局视角、决策依赖主观印象 |

### 3.2 模块功能需求明细

#### 模块 0：Agentic Learning Engine（智能学习引擎）

| 功能点 | 需求描述 | 优先级 | 验收标准 |
|--------|---------|--------|---------|
| **Orchestrator 事件调度** | 接收学员行为/工作流/时间/状态变化/外部事件，查询 Memory，决策 Agent 执行计划 | P0 | 支持10+事件类型，决策延迟<3s |
| **Orchestrator 冲突仲裁** | 当多个 Agent 建议冲突时，按优先级规则仲裁（如预测Agent疲劳预警 > 内容Agent内容推送） | P0 | 冲突解决准确率>85% |
| **Memory 热记忆（Hot）** | 冻结快照：USER.md（学员画像）+ MEMORY.md（情境记忆），每次推理自动注入 | P0 | 体积~1500 tokens，Nudge每30分钟更新 |
| **Memory 冷回忆（Cold）** | 会话档案馆：Agent-学员对话、决策日志、行为埋点，支持全文搜索+语义检索 | P0 | 按需检索，不自动注入上下文 |
| **Memory 程序性记忆** | Skill Documents：每次成功干预后自动合成策略文档，下次复用并持续改进 | P1 | 技能文档成功率可追踪，版本化管理 |
| **Memory 外部知识** | LLM Wiki + 课程库 + 案例库 + 资源库 + 导师网络，所有 Agent 共享 | P0 | 支持RAG检索，自动索引更新 |
| **Tool Registry** | 标准化工具目录：内容/测评/通讯/项目/分析五大类工具，含输入/输出/权限/回退机制 | P0 | 所有工具可通过标准化接口调用 |
| **诊断 Agent** | 作品分析 + 微场景测试 → 生成能力缺口热力图 + 个性化新手村路径 | P0 | 作品分析准确率>70%，导师认可度>80% |
| **Coach Agent** | 苏格拉底式对话、实时纠错、引导式复盘，7×24在线 | P0 | 处理80%基础提问，思考深度可评估 |
| **内容 Agent** | 根据学员状态动态组装学习套餐（课程+案例+微实操+对话） | P0 | 路径完成率>60%，千人千面 |
| **预测 Agent** | 监控学习行为数据，识别风险信号（卡顿/流失/疲劳），早期预警 | P1 | 预警命中率>60%，误报率<30% |
| **实战 Agent** | 嵌入工作流（PR/AE/飞书侧边栏），上下文感知即时指导 | P2 | 推送采纳率>50% |
| **AI Assessment Generator** | 自动分析课程内容 → 生成测试题 → Pending Pool → Owner审阅 → 发布 | P1 | Owner审阅通过率>70%，审阅耗时<10分钟 |

## 四、非功能需求

### 4.1 性能需求

| 指标 | 需求描述 | 目标值 |
|------|---------|--------|
| **Orchestrator 决策延迟** | 从接收事件到输出 Agent 执行计划的平均时间 | **< 3秒** |
| **Agent 消息响应延迟** | Coach Agent 对话响应、内容 Agent 推送的端到端延迟 | **< 5秒** |
| **规则引擎响应** | 80%简单场景由规则引擎处理（不涉及 LLM 推理） | **< 100ms** |
| **LLM 推理响应** | 20%复杂场景由 LLM 处理 | **< 3秒** |
| **系统并发** | 支持全公司100人同时使用，高峰期（周一上午）50人并发 | 无感知卡顿 |
| **数据查询** | 能力图谱/学习记录查询响应 | **< 1秒** |
| **视频加载** | 课程视频首帧加载时间 | **< 2秒** |

### 4.2 安全需求

| 需求项 | 具体描述 |
|--------|---------|
| **最小权限原则** | 不同 Agent 有不同的工具调用权限；预测 Agent 不能直接发送消息，需 Orchestrator 调度 |
| **数据脱敏** | Agent 决策日志中隐藏学员真实姓名；敏感数据不出境 |
| **作品数据隔离** | 学员作品分析数据仅限培训系统使用，不与绩效考核直接挂钩 |
| **调用审计** | 所有工具调用记录到 Memory，确保可审计、可追溯 |
| **内容安全** | AI 生成内容（测试题/学习推荐）需经过置信度校验，低置信度标记为「待核实」 |
| **访问控制** | 学员只能查看自己的记忆数据；导师可查看所带学员；管理层可查看团队汇总 |

### 4.3 可用性需求

| 需求项 | 具体描述 |
|--------|---------|
| **系统可用性** | 核心业务（学习推送/能力图谱/测评）可用性 **≥ 99.5%** |
| **降级模式** | LLM 不可用时，规则引擎接管关键功能（推送固定路径、基础预警） |
| **飞书集成优先** | 优先使用飞书原生能力（多维表格/云文档/智能伙伴），减少外部依赖 |
| **多终端适配** | 飞书消息（手机/PC）、飞书文档、飞书应用引擎页面全覆盖 |

### 4.4 可扩展性需求

| 需求项 | 具体描述 |
|--------|---------|
| **岗位扩展** | 系统架构支持二期扩展摄影师/制片等岗位的能力图谱 |
| **Agent 扩展** | Orchestrator 支持新增 Agent 类型，无需重构核心调度逻辑 |
| **工具扩展** | Tool Registry 支持动态注册新工具，遵循标准化接口规范 |
| **内容扩展** | 课程库/案例库/资源库支持无上限增长，检索性能不降级 |
| **用户规模扩展** | 架构支持从100人扩展至500人，关键组件可水平扩展 |

### 4.5 可维护性需求

| 需求项 | 具体描述 |
|--------|---------|
| **Agent 决策可解释** | 每次推荐必须说明理由（"推荐这节课，因为你在上周测评中节奏感得分L1.5"） |
| **决策日志审查** | 管理端可查看各 Agent 决策日志，确保透明度 |
| **Prompt 版本管理** | 各 Agent 的系统提示（System Prompt）需版本化，支持A/B测试与回滚 |
| **Skill Doc 自改进** | 程序性记忆自动积累，成功干预经验自动沉淀为组织资产 |

## 五、业务需求与验收指标（KPI）

### 5.1 核心业务指标

| 指标类别 | 指标名称 | 计算方式 | 目标值 | 测量周期 |
|---------|---------|---------|--------|---------|
| **学习效率** | 新人独立产出周期 | 从入职到首次独立交付项目的天数 | **≤14天** | 每新人 |
| **学习效率** | 课程完成率 | 已完成课程数 / 分配课程数 | **≥90%** | 每周 |
| **学习效率** | 平均学习时长 | 总学习时长 / 活跃用户数 | **≥30分钟/天** | 每周 |
| **能力成长** | 新人达标时间 | 入职到首次独立产出天数 | **<14天** | 每新人 |
| **能力成长** | 能力评估覆盖率 | 已评估人数 / 总人数 | **100%** | 每月 |
| **人才保留** | 项目返工率 | 返工项目数 / 总项目数 | **<10%** | 每月 |
| **人才保留** | 人才保留率 | 1年后仍在职人数 / 入职人数 | **>90%** | 每年 |
| **导师效率** | 导师参与率 | 参与带教人数 / 资深员工数 | **>60%** | 每季度 |
| **导师效率** | 学员满意度 | 学员对导师评分平均分 | **>4.5/5** | 每季度 |

### 5.2 Agent 效能指标

| 指标类别 | 指标名称 | 计算方式 | 目标值 | 测量周期 |
|---------|---------|---------|--------|---------|
| **Agent 效率** | Agent 自主解决率 | Agent 独立完成的事件数 / 总事件数 | **>70%** | 每周 |
| **Agent 效率** | 导师时间节省率 | (v1.0耗时 - v2.0耗时) / v1.0耗时 | **>50%** | 每月 |
| **Agent 效率** | 平均决策延迟 | Orchestrator 接收事件到输出决策的平均时间 | **<3秒** | 实时 |
| **Agent 效率** | 人机交接准确率 | Agent 判断"需要人工"的准确率 | **>85%** | 每月 |
| **Agent 质量** | 学员对 Agent 满意度 | 学员对推送/对话/干预的满意度评分 | **>4.0/5** | 每月 |
| **Agent 质量** | 预测 Agent 预警命中率 | 预警后实际发生问题的比例 | **>75%** | 每月 |
| **Agent 质量** | 内容 Agent 路径完成率 | Agent 推荐的学习路径的实际完成率 | **>80%** | 每周 |

### 5.3 业务验收标准（Acceptance Criteria）

#### P0 功能验收

- [ ] 新人收到 Offer 后48小时内，诊断 Agent 自动发起作品收集
- [ ] 入职当天，学员在飞书中收到个性化新手村路径（非统一模板）
- [ ] 每日任务通过飞书消息+待办推送，学员无需主动打开系统
- [ ] Agent 自动感知任务完成状态（通过内容消费埋点、作业提交检测）
- [ ] Day 7完成率>90%，且毕业报告包含能力缺口热力图+下一阶段路径建议
- [ ] 每门课程有明确的学习目标（Learning Objective）
- [ ] 案例拆解包含至少6个维度（选题/脚本/拍摄/剪辑/包装/数据）

#### P1 功能验收

- [ ] 输入项目类型+风格+难度，系统推荐3-5位最合适员工并显示匹配理由
- [ ] AI 生成测试题经 Owner 审阅后发布，审阅平均耗时<10分钟
- [ ] 导师每周处理基础问题时间从4小时降至2小时
- [ ] 预警命中率>60%，误报率<30%
- [ ] 能力雷达图支持实时更新，Agent 自动维护
